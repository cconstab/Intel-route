## Architecture Principles
- Each person should have their own Atsign (e.g. @alice, @bob) — the core identity unit.
- Agents/apps can have a dedicated Atsign OR share one using namespaces for isolation.
- Multiple instances of the same agent can run under a single Atsign (see Multi-Agent Coordination).
- All data is end-to-end encrypted by default; only the Atsign owner holds the keys.
- There is NO application backend. atServers are personal data stores + rendezvous points.
- Use namespaces to separate app data (`key.myapp@atsign`).
- Share with `AtKey..sharedWith`; only use `metadata.isPublic = true` for intentionally public data.
- Do NOT store application data in local files — store as AtKeys so it syncs across devices.

## Initialization Pattern (Dart CLI / Agents)
- `AtClientManager.getInstance().setCurrentAtSign(atSign, namespace, preferences)`.
- `AtClientPreference` with `rootDomain`, `hiveStoragePath`, `commitLogPath`, `namespace`.
- Each agent/instance on one machine MUST use a unique `hiveStoragePath` + `commitLogPath`.
- For CLI/agent auth use `at_cli_commons` `CLIBase.fromCommandLineArgs(args)`.
- (Python `atsdk`: `AtClient(atSign, root_address=Address.from_string(host:port), queue=Q)`;
  always pass `root_address` — the default is production.)

### Welcome / Auth Screen (four workflows)
1. **Login from Keychain** — `KeychainStorage().getAllAtsigns()` → `AtSignSelectionDialog`
   → `AtAuthRequest(atSign, atKeysIo: KeychainAtKeysIo())` → `PkamDialog` → `setCurrentAtSign()`.
2. **Onboard a New Atsign (Registrar)** — follow `onboard()` in the example: selection →
   `RegistrarCramDialog` → `CramDialog` → setup. `RegistrarService(registrarUrl: "my.atsign.com", apiKey: ...)`.
3. **APKAM enrollment (new device)** — `ApkamActivationDialog.show(...)` → approve on an
   authorized device → `AtAuthRequest(atSign, atAuthKeys: ...)` → `PkamDialog` → setup.
4. **Login via .atKeys file** — `AtKeysFileDialog.show()` → `FileAtKeysIo` → extract atSign
   from filename → `AtAuthRequest(atSign, atKeysIo: fileAtKeysIo)` → `PkamDialog(backupKeys:[KeychainAtKeysIo()])` → setup.

Post-auth: app support dir → `AtClientPreference(rootDomain, namespace, hiveStoragePath,
commitLogPath)` → `setCurrentAtSign(atSign, namespace, prefs, atChops:.., atLookUp:..)`.

## Communication Patterns
1. **Async (fire-and-forget):** `atClient.put()` / `get()`; `PutRequestOptions()..useRemoteAtServer = true` to write to the cloud secondary directly.
2. **Sync (delivery confirmation):** `notificationService.notify(NotificationParams.forUpdate(key, value:))` → `NotificationResult`.
3. **Notification / event-driven:** receiver `notificationService.subscribe(regex:"pattern", shouldDecrypt:true).listen(...)`; sender `notificationService.notify(...)`. Auto-reconnect; regex filtering. *(This project's core pattern.)*
4. **RPC:** `AtRpc` / `AtRpcClient` with `AtRpcCallbacks` + allowList. *(Policy plane.)*
5. **Streaming:** `subscribe(regex:"stream.*")` + `notify()` in a loop; bulk files via encrypted upload + shared URL/key.
6. **TCP / sockets (NoPorts):** `noports_core` / `npt` for E2E-encrypted TCP with zero open ports. *(Path A alternative.)*

## Data Storage and Synchronization
- `atClient.put()` stores an AtKey locally and auto-syncs to the atServer.
- NEVER store application data in local files — store as AtKeys (syncs across devices).
- Namespaces separate app data; `sharedWith` shares; `isPublic` only for public data.

## Multi-Agent Coordination
- Multiple instances under one Atsign: (1) **mutex race** with immutable keys
  (`Metadata()..immutable = true`, `useRemoteAtServer = true`, TTL to auto-release); or
  (2) **`ServiceFactoryWithNoOpSyncService()`** with ephemeral per-session storage.

## Networking, File Transfer & Access Control
- **TCP tunnels:** NoPorts (`noports_core`) for port-less encrypted TCP.
- **Encrypted file transfer:** encrypt client-side, upload to a file service, share URL+key as an AtKey.
- **Policy enforcement:** forward authorization decisions to a policy atSign (the policy plane);
  default-deny; rules by identity + role + action.

## Error Handling
- Wrap AtClient operations in try/catch. Common: `AtClientException`, `AtKeyException`, `AtConnectException`.
- Check `syncService.syncInProgress` before critical operations.

## Key Conventions
- Namespace per app (`smartroute` here). Validate Atsigns with at_commons `'@bob'.toAtsign()` (Dart).
- Identity, encryption, and key custody are platform guarantees — not application nodes.
